# Cloudflare /crawl API - Complete Details

## Request Parameters

**Required:**
- `url` (string) - Starting URL to crawl

**Key Optional Parameters:**
- `limit` - Max pages to crawl (default: 10, max: 100,000)                                                              - `depth` - Max link depth (default: 100,000)
- `formats` - Response type: HTML, Markdown, or JSON                                                                    - `render` - Boolean; execute JavaScript (default: true)
- `source` - URL discovery source: "all", "sitemaps", or "links"
- `maxAge` - Cache validity in seconds (default: 86,400)
- `modifiedSince` - Unix timestamp for selective crawling
- `options.includeExternalLinks` - Follow external domains (default: false)
- `options.includeSubdomains` - Follow subdomains (default: false)
- `options.includePatterns` / `excludePatterns` - Wildcard URL filters
- `authenticate` - HTTP credentials
- `setExtraHTTPHeaders` - Custom headers for token auth
- `gotoOptions` - Page load behavior settings
- `rejectResourceTypes` - Block resource types (e.g., images, media)

## Response Format

Jobs return asynchronously with a `job_id`. Results include:
- `status` - Current state (running, completed, errored, cancelled_due_to_timeout, etc.)
- `browserSecondsUsed` - Rendering time consumed
- `total` / `finished` - Page counts
- `records` - Array containing per-URL data with markdown, HTML, metadata, and HTTP status
- `cursor` - Pagination token if results exceed 10 MB

## Limitations

**Free Plan Restrictions:** "Users on the Workers Free plan are subject to additional crawl-specific restrictions."

**Timeouts:** Jobs have 7-day maximum runtime; results persist 14 days after completion.

**Rate Limiting:** The crawler implements backoff and respects `robots.txt` crawl-delay directives.

**Resource Consumption:**
- Free tier: 10 minutes browser time daily
- `render: true` uses headless browser billing
- `render: false` currently unbilled (beta phase)

## robots.txt & Bot Identification

The crawler "respects the directives of `robots.txt` files" and identifies as
`CloudflareBrowserRenderingCrawler/1.0`. Blocked URLs appear with `"status": "disallowed"`.

**Critical Limitation:** "Browser Rendering does not bypass CAPTCHAs, Turnstile challenges, or any other bot
protection mechanisms." Bot protection rules (Bot Management, WAF, Turnstile) apply equally to crawls. However,
operators can create WAF skip rules to allowlist Browser Rendering on their own sites.

## Crawling Restrictions

The crawler cannot access content behind active bot protections unless explicitly allowlisted. No method exists to
bypass these defenses—the service applies the same security rules as standard traffic.
