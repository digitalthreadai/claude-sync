How it works

Three phases. One run.

founder-docs doesn't ask questions or make assumptions. It reads your codebase, then writes.

01

Discover

Reads package.json, schema files,
env examples, route files,
config files, and components.
Builds a complete picture first.

Routes, tables, env vars, dependencies, brand colors — all extracted before writing a single word.

02

Write

Generates strategy.md,
setup.md, and README.md
from the discovered data.
Skips irrelevant sections.

Templates adapt to what exists. No revenue section if there's no monetization. No roadmap if none found.

03

Publish

Generates 4 self-contained
HTML pages in docs/.
No build step. No dependencies.
Open and ship.

Dark-themed, mobile-responsive, interactive. Brand colors applied if detectable from your CSS/theme config.

What does discovery actually read? +

Package metadata: package.json, requirements.txt, Cargo.toml — all deps with versions.

Routes and pages: src/app/**/page.tsx, pages/**/*.tsx, routes/**/* — grouped public / admin / API.

Database: prisma/schema.prisma, supabase/migrations/*, drizzle/**/* — tables, columns, relationships.

Environment: .env.example, .env.local.example — every variable with its description.

Config and auth: next.config.*, tailwind.config.*, vercel.json — deployment target, auth provider, brand colors.

Install

One line.

git clone https://github.com/digitalthreadai/claude-sync.git /tmp/cs && cp -r /tmp/cs/skills/founder-docs ~/.claude/skills/

Then restart Claude Code and run: use founder-docs from any project directory.

The real value

Most projects are underdocumented — not because founders are lazy, but because writing docs while building is a context switch that never happens. founder-docs makes it a 60-second step instead of a 3-hour project.

Run it before you onboard your first contributor, before you raise a round, before you step away for a week. The docs are always current because they're always generated fresh.
